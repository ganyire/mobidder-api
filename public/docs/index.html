Introduction

Mobidder API documentation

Base URL: http://127.0.0.1:8000

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer 38|NCg3209E9muRYFLCqqlBOs9cFll3yrcgSBxw0IqQ".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

AUTH MANAGEMENT

APIs for managing authentication

Register

Endpoint for registering a new user.

Example request:
const url = new URL(
    "http://127.0.0.1:8000/api/v1/auth/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "name": "Prof. Retha Hegmann II",
    "email": "hill.kathleen@johns.com",
    "password": "Lenny@007",
    "password_confirmation": "Lenny@007",
    "phone": "+13514607966",
    "role": "vendor-admin",
    "street": "1234 Main St",
    "city": "Nairobi",
    "state": "Nairobi",
    "zip_code": "00100",
    "country": "Kenya"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
$client = new \GuzzleHttp\Client();
$response = $client->post(
    'http://127.0.0.1:8000/api/v1/auth/register',
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [

            'email' => 'hill.kathleen@johns.com',
            'password' => 'Lenny@007',
            'password_confirmation' => 'Lenny@007',

            'street' => '1234 Main St',
            'city' => 'Nairobi',

Headers

Example: application/json

Body Parameters

The name of the user. Must not be greater than 255 characters. Example: Prof. Retha Hegmann II

The password of the user. Example: Lenny@007

password_confirmation   string   

The password confirmation of the user. Example: Lenny@007

The phone number of the user. Must not be greater than 255 characters. Example: +13514607966

role   string   

The role of the user. Must be one of super-admin, vendor, customer, or vendor-admin Must not be greater than 255 characters. Example: vendor-admin

street   string  optional  

The street of the user. Example: 1234 Main St

city   string  optional  

The city of the user. This field is required when street is present. Example: Nairobi

state   string  optional  

The state of the user. Example: Nairobi

zip_code   string  optional  

The zip code of the user. Example: 00100

country   string  optional  

The country of the user. This field is required when street is present. Example: Kenya
